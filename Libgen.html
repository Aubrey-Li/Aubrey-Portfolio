Usability Analysis


According to Nielsen Norman Group, Usability is a quality attribute that assesses how easy user interfaces are to use. The word "usability" also refers to methods for improving ease-of-use during the design process.


There are 5 core quality components to usability:




Learnability

The current UI is relatively easy to learn for a specific group of users--students, scholars, or book lovers who want to get access to free public recourses and increase their knowledge, if s/he comes with purpose of finding a very specific book. If the user doesn't have a book or article in mind, it is rather unlikely that the user would visit this website, since it is not easy to browse book options and new release on this website compared to sites such as Goodreads and Amazon.




Efficiency

The website should be pretty efficient for users who have a clear goal of finding a book, but less so for those who want to explore the website and see what they have to offer, since there are too many information listed without a clear structure as mentioned in the learnability section.




Memorability

The front page is pretty memorable, since old users of the Libgen website are rather unlikely to forget the steps to get access to their desired books or articles. However, old users are unlikely to explore other parts of the front page, since the webpage is rather disorganized and it is hard to tell the hierarchy between different links and dropdowns.




Satisfaction

The user satisfaction of the website should be moderate. However, satisfaction rate might decrease due to convolusion in operating the front page and the general searching experience. The font of the website is also very small for readers so that when doing searches.




Few Errors

In general, the website doesn't have too many errors, but sometimes the links can lead users to very confusing pages. However, the amount of errors or indistinguishable text through clicking some link would leave users confused and the user experience isn't optimal.

From Low-Fidelity to High-Fidelity

The process from Low-fidelity wireframing to High-fidelity prototyping is a process of evaluating what's aesthetic, what's not; It's a process of adding more possibility of interaction to see which elements in a low-fi wireframe is unreasonable. In this process, I decided upon a rather harmonious while not boring color palette to show consistency but also contrast when necessary. I categorized elements so they belong to different groups to create a visual hierarchy. I used rather consistent fonts to add to the hierarchy. Ultimately, I used this layout to emphasize important information to increase efficiency. There are changes from the low-fi to high-fi structure, as I thought the latter more aesthetic and creates more spatial dynamics.

Final Comments & Observations

Hope you enjoy using the redesigned Libgen website! In the final responsive redesign, I decided to make the break points for navbar smaller so that tablets could also have a holistic view of all the navbar items. I make this decision based on the fact that many tablets serve the same function as laptops for many people today. Since the functionality are similar, the design of the navbar should resemble one another. In addition, to increase consistency for all the cards, I added "Read More" option to the "Alternative Domains" card. The design of the dropdowns are more modern and that in the prototype. Using the developer tools, the screen could be seen adjusted to fit different device sizes, though for 4K, the content would gather towards the upper part of the screen, as the website is designed to be a one-pager.
